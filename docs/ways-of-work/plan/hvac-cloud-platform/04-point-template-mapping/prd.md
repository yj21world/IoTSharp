# Feature PRD：设备大类与点位模板应用

## 1. Feature 名称

设备大类与点位模板应用

## 2. 所属 Epic

- 父级产品需求：[暖通空调云端物联网平台长期产品需求](../epic.md)
- 父级架构文档：[暖通空调云端物联网平台架构规格](../arch.md)
- 前置模块：[网关与设备接入模块](../01-gateway-device-access/prd.md)、[透传网关 Modbus 采集模块](../02-transparent-modbus-collection/prd.md)

## 3. 目标

### 问题

当前系统中设备分类语义容易被混用：历史代码曾把普通设备/网关当作设备自身属性，规划文档又用模板表达水泵、主机、电表等业务分类和点位模板。这会让新增设备、网关接入、采集模板和业务分类混在一起，导致后续扩展变复杂。暖通项目中真正需要的是“设备大类”能力：同一大类设备可以复用默认点位模板，但设备实例本身仍然只是具体物理设备的链接锚点。

### 解决方案

将当前规划中的模板分类收敛为“设备大类”。设备可以选择关联一个设备大类；只有在设备关联设备大类时，系统才触发该大类相关的点位模板应用逻辑。未关联设备大类的设备保持普通设备行为，不自动生成采集点位、不修改采集任务、不触发模板校验。

### 影响

该功能减少现场重复配置，同时避免把模板逻辑强加给所有设备。调试工程师可以为同类设备维护一套点位模板，并在需要时显式应用到具体设备；普通设备或临时设备可以不关联设备大类，继续按手工配置或现有流程运行。

## 4. 使用对象

- 调试工程师：创建设备大类，维护同类设备的默认点位模板。
- 项目实施人员：新增设备时选择是否关联设备大类。
- 平台管理员：维护设备大类清单，控制模板是否启用。
- 后续开发 agent：基于明确边界实现模板应用，不改动无设备大类的设备行为。

## 5. 用户故事

- 作为调试工程师，我希望创建设备大类并配置默认 Modbus 点位模板，以便同类设备快速复用采集配置。
- 作为项目实施人员，我希望新增设备时可以不选择设备大类，以便临时设备或特殊设备不触发模板逻辑。
- 作为项目实施人员，我希望给设备关联设备大类后再显式应用模板，以便系统自动生成大部分采集点位。
- 作为调试工程师，我希望重复应用模板时系统能跳过或更新已有点位，以便避免重复点位。
- 作为平台管理员，我希望禁用设备大类后不能继续应用模板，以便控制错误模板扩散。

## 6. 需求

### 6.1 功能需求

- 系统必须引入“设备大类”概念，用于替代当前文档中的业务分类语义。
- 删除公开的设备角色字段、枚举和 API 参数，不再把普通设备/网关作为设备自身属性暴露。
- 若 EF 继承映射仍需要区分 `Device` / `Gateway`，只能使用内部判别列（如 `DeviceRole`），不得进入 DTO、前端表单或业务分类逻辑。
- 设备大类必须支持创建、编辑、删除、启用、禁用。
- 设备大类至少包含：大类标识、大类名称、暖通分类、描述、启用状态、版本号。
- 设备可以不关联设备大类。
- 设备未关联设备大类时，系统不得自动触发点位模板、采集任务生成、模板校验或其它设备大类逻辑。
- 设备关联设备大类后，系统应记录关联关系。
- 设备大类必须支持维护 Modbus 点位模板。
- 点位模板至少包含：点位 key、显示名、单位、目标类型、值类型、功能码、寄存器地址、寄存器数量、字节序、字顺序、轮询周期、转换规则。
- 系统必须支持将设备大类模板应用到设备。
- 模板应用时必须生成或同步 `CollectionTask`、`CollectionDevice`、`CollectionPoint` 相关记录。
- 模板应用时必须支持处理已有同名点位，第一阶段默认策略为按 `PointKey` 幂等更新，不重复创建。
- 模板应用不得将点位明细写入 `Device` 表。
- 模板应用失败时不得留下半完成点位，必须使用事务保证一致性。
- 删除设备大类前必须检查是否已有设备关联；有关联时第一阶段阻止删除。
- 禁用设备大类后，已关联设备不自动解除关联，但不得继续应用模板。
- 所有新增/调整 API 必须使用 `ApiResult<T>`，列表/查询返回 `{ total, rows }`。

### 6.2 非功能需求

- 设备大类逻辑必须是显式触发，不得影响未关联设备大类的现有设备。
- 模板应用必须具备幂等性，重复执行不会生成重复点位。
- 模板应用必须记录可排查的错误信息。
- 第一阶段不引入自动级联更新，模板变更不会自动影响已应用设备。
- 第一阶段不处理边缘网关 JSON 字段映射。
- 第一阶段不实现告警模板、规则模板、控制模板和链接模板。
- 前端必须使用 Vue 3 + Element Plus 显式组件，不使用 `fs-crud` / `@fast-crud/fast-crud`。

## 7. 验收标准

### 7.1 设备大类管理

- Given 用户已登录且有权限
  When 创建设备大类“西门子 V20 变频器”
  Then 系统保存设备大类并返回 `{ code, msg, data: { total: 1, rows: [...] } }`。

- Given 设备大类已被设备关联
  When 用户删除该设备大类
  Then 系统阻止删除并返回空分页对象。

### 7.2 设备可选关联

- Given 新建设备未选择设备大类
  When 保存设备
  Then 设备创建成功，且不生成 `CollectionTask`、`CollectionDevice`、`CollectionPoint`。

- Given 设备已存在但未关联设备大类
  When 查询设备详情
  Then 系统显示设备大类为空，不触发模板逻辑。

### 7.3 模板应用

- Given 设备已关联启用状态的设备大类，且大类包含 5 个点位模板
  When 用户执行“应用模板”
  Then 系统生成或同步对应 `CollectionTask`、`CollectionDevice`、`CollectionPoint`。

- Given 同一设备已应用过模板
  When 用户再次执行“应用模板”
  Then 系统按 `PointKey` 幂等更新，不重复创建点位。

- Given 设备未关联设备大类
  When 用户执行“应用模板”
  Then 系统返回明确错误，不创建任何点位。

- Given 设备关联的设备大类已禁用
  When 用户执行“应用模板”
  Then 系统返回明确错误，不创建任何点位。

## 8. 不在范围内

- 不在本功能中重写网关接入模型；网关仍可作为 `Gateway : Device` 派生类型存在。
- 不在本功能中实现边缘网关 JSON 映射。
- 不在本功能中实现告警模板、规则模板、控制模板、链接模板。
- 不在本功能中实现模板版本历史和自动级联同步。
- 不在本功能中重构 Produce 产品模型。

## 9. 当前状态判断

当前代码已有 `DeviceCategory`、`CollectionRuleTemplate`、`DeviceCategoryController`、`DeviceCategoryService.ApplyProfileToDeviceAsync`。第一阶段可以复用这些代码作为“设备大类”的技术承载，但产品文案、API 语义和前端页面应统一显示为“设备大类”，并确保设备未关联设备大类时不触发模板应用逻辑。
