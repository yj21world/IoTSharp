# 全局设计笔记：Device 锚点与设备大类

> 创建日期：2026-04-28  
> 状态：设计原则，当前阶段仅落地“设备大类 + 点位模板应用”

## 1. 核心原则

`Device` 应只作为“平台与具体物理设备之间的链接定义”。它是现场设备在平台中的稳定锚点，负责表达这个对象是谁、属于哪个租户/客户、如何被识别，以及它与真实物理设备的一一或近似一一对应关系。

除最小身份和归属信息外，采集点位和后续其它能力不应继续堆到 `Device` 表中，而应通过独立领域表关联 `DeviceId`。

当前阶段先缩小范围：只处理“设备大类”和“设备大类下的点位模板应用”。属性、告警、链接、规则、控制和能耗能力仅作为后续扩展方向，不在本阶段实现，也不在本阶段预留复杂模板聚合。

## 2. Device 应表达什么

`Device` 的长期职责：

- 设备唯一标识：`Id`、`Name`、外部编号或现场编号。
- 归属边界：`Tenant`、`Customer`。
- 设备接入身份：通过 `DeviceIdentity` 关联。
- 与具体物理设备的链接：例如现场设备编码、安装位置、铭牌信息或外部系统 ID。
- 少量稳定状态：如软删除标记、创建时间等。

`Device` 不应承担：

- 采集点位明细。
- 采集任务编排。
- 告警规则。
- 控制命令模板。
- 规则链绑定。
- 能耗统计配置。
- 网关连接拓扑的复杂语义。
- 暖通业务大类的完整模板定义。

## 3. 能力通过独立表关联 Device

推荐长期关系：

```text
Device
├── DeviceIdentity              // 接入身份
├── CollectionTask / Point       // 采集任务与点位
└── 后续扩展表                  // 告警、链接、规则、控制、能耗等
```

这样做的目标是让 `Device` 保持稳定，不因为新增采集方式、告警类型、控制策略或统计模型而频繁膨胀。

## 4. 设备大类

在 `Device` 之上，需要新增“设备大类”概念，用于描述同一类设备的业务分类，并在当前阶段承载默认 Modbus 点位配置。

设备大类当前阶段只回答一个问题：

> 这一类设备通常有哪些默认 Modbus 采集点位？

示例：

- 西门子 V20 变频器模板。
- 冷冻水泵模板。
- 冷水机组模板。
- 电表模板。
- 温湿度传感器模板。

设备大类当前阶段包含：

- 大类标识。
- 大类名称。
- 暖通分类。
- 描述、版本、启用状态。
- Modbus 点位模板：点位 key、寄存器、功能码、数据类型、单位、转换规则等。

属性、告警、链接、规则、控制、能耗等能力暂不进入本阶段设备大类模型。

## 5. 模板应用到设备

模板应用到设备时，不应把模板内容复制进 `Device` 表，而应生成或绑定对应的独立配置记录。

```text
设备大类
   └── 点位模板       ──应用──> CollectionTask / CollectionDevice / CollectionPoint
```

应用策略必须明确：

- 首次应用：生成默认配置。
- 重复应用：跳过、覆盖或同步差异，策略要可见。
- 现场覆盖：允许设备实例保留少量差异，如 SlaveId、采集周期、禁用点位。
- 模板变更：第一阶段不自动级联更新已应用设备，后续可设计显式同步。
- 未关联设备大类：不触发任何模板应用逻辑。

## 6. 与现有代码的映射

公开模型不再保留普通设备/网关角色字段。普通设备/网关不应作为 `Device` 的业务属性；如 EF 继承映射需要区分 `Device` / `Gateway`，只能使用内部判别列（如 `DeviceRole`），并通过类型判断或 `_context.Gateway` 查询使用。

当前代码已有的 `DeviceCategory` 服务/控制器和 `CollectionRuleTemplate` 可作为设备大类的第一阶段过渡承载：

- `DeviceCategories`：设备大类物理表。
- `DeviceCategory`：短期保留为代码层过渡命名，后续可重命名为 `DeviceCategory`。
- `CollectionRuleTemplate`：承载采集点位模板。
- `Device.DeviceCategoryId`：记录设备应用的大类模板。

后续可逐步演进为更清晰的命名，例如 `DeviceCategory`、`DeviceCategoryTemplate`、`CollectionTemplate` 等。

## 7. 设计约束

- 不关联设备大类时，不触发设备大类相关逻辑。
- 新增字段前优先判断是否应放入独立表，而不是 `Device`。
- 设备大类是分类与默认点位配置入口，不是具体设备。
- `Device` 是设备实例锚点，不是模板或采集任务容器。
- 当前阶段不考虑边缘网关 JSON 模板。
- 当前阶段不实现属性、告警、链接、规则、控制和能耗能力。
