# Feature PRD：透传网关 Modbus 采集模块

## 1. Feature 名称

透传网关 Modbus 采集模块

## 2. 所属 Epic

- 父级产品需求：[暖通空调云端物联网平台长期产品需求](../epic.md)
- 父级架构文档：[暖通空调云端物联网平台架构规格](../arch.md)
- 前置模块：[网关与设备接入模块](../gateway-device-access/prd.md)

## 3. 目标

### 问题

暖通现场存在大量只支持 Modbus RTU/TCP 的设备，例如水泵变频器、冷水机组、电表、阀门、温湿度传感器和压力传感器。对于 USR-G805 这类透传网关，网关只负责 MQTT 与串口数据之间的转发，不具备主动轮询、点位解析、边缘计算和业务映射能力。因此云端必须承担采集任务配置、Modbus 请求生成、响应匹配、数据解析、换算、入库和诊断日志等职责。

### 解决方案

在现有 IoTSharp 接入能力之上，建设云端 Modbus 采集模块。平台通过采集任务描述网关、从站、点位、轮询周期、寄存器、数据类型、字节序、转换规则和目标遥测键；后台采集服务按任务调度生成 Modbus 请求帧，通过 MQTT 下发给透传网关；网关返回响应后，云端匹配请求、校验响应、解析数据、应用换算规则，并写入最新值和历史遥测链路。

### 影响

该模块是暖通云端平台的核心业务闭环之一。完成后，平台可以在不依赖边缘采集能力的情况下接入现场 Modbus 设备，并为后续实时数据、历史曲线、设备聚合、告警规则、控制命令和能耗统计提供稳定数据来源。

## 4. 使用场景

- 现场使用透传网关，网关通过 MQTT 连接云端。
- 一个网关下挂多个 Modbus 从站设备。
- 一个从站设备包含多个寄存器点位。
- 不同点位具有不同轮询周期、数据类型、倍率、偏移量和目标遥测键。
- 调试人员需要查看每次请求帧、响应帧、解析结果、错误原因和超时情况。

## 5. 用户故事

- 作为现场调试人员，我希望能为一个透传网关配置 Modbus 采集任务，以便云端按配置轮询下级设备。
- 作为现场调试人员，我希望能配置从站地址、功能码、寄存器地址、长度、数据类型和字节序，以便正确读取不同厂商设备的数据。
- 作为现场调试人员，我希望能配置倍率、偏移量和枚举映射，以便把原始寄存器值转换为可读业务值。
- 作为运维人员，我希望采集成功后能在设备实时数据和历史数据中看到对应遥测值。
- 作为调试人员，我希望采集失败时可以查看请求帧、响应帧、错误状态和耗时，以便定位现场接线、网关、地址或协议问题。
- 作为平台维护人员，我希望采集任务启停和配置变更能被后台服务自动识别，以便减少重启服务的次数。

## 6. 需求

### 6.1 功能需求

- 系统必须支持按网关创建设备采集任务。
- 采集任务必须包含任务标识、关联网关、协议类型、启用状态、版本、连接配置和上报策略。
- 系统必须支持在采集任务下配置多个 Modbus 从站设备。
- 从站设备必须包含从站标识、从站名称、SlaveId、启用状态、排序和协议扩展配置。
- 系统必须支持在从站下配置多个采集点位。
- 点位必须支持功能码 `1/2/3/4`，分别对应线圈、离散输入、保持寄存器和输入寄存器读取。
- 点位必须支持寄存器地址、寄存器数量、原始数据类型、字节序、字顺序、轮询周期和轮询分组。
- 点位必须支持数值转换规则，例如倍率、偏移量、布尔映射和枚举映射。
- 点位必须支持映射到目标设备的遥测、属性或告警输入，首期优先遥测。
- 后台采集服务必须按启用任务加载配置，并按点位轮询周期进行调度。
- 系统必须在发送请求前检查网关在线状态，离线网关不得持续发送采集请求。
- 系统必须通过 MQTT 向透传网关发送 Modbus 请求帧。
- 系统必须通过 MQTT 接收网关返回的 Modbus 响应帧。
- 系统必须能通过 requestId 或等效机制匹配请求与响应。
- 系统必须校验响应帧，识别成功、超时、CRC 错误、异常响应、解析失败等状态。
- 成功响应必须解析为目标值，并发布到现有遥测链路。
- 系统必须记录采集日志，包括网关、任务、从站、点位、请求帧、响应帧、状态、错误信息和耗时。
- 系统必须提供采集任务 CRUD、启用、禁用、校验、预览和日志查询 API。
- 新增/调整 API 必须遵循 `ApiResult<T>`，列表/查询结果使用 `{ total, rows }`。

### 6.2 当前已具备能力

- 已存在 `CollectionTask`、`CollectionDevice`、`CollectionPoint`、`CollectionLog` 实体。
- 已存在 `CollectionTaskController`，支持采集任务 CRUD、启停、草稿、校验、预览和日志查询。
- 已存在 `CollectionTaskService`，支持任务与 DTO 映射、持久化和日志查询。
- 已存在 `ModbusCollectionService`，作为后台采集服务入口。
- 已存在 `GatewayScheduler`、`GatewaySchedulerManager`、`BatchMerger`，支持按网关和点位周期调度。
- 已存在 `ModbusRtuProtocol` 和 `ModbusDataParser`，支持组帧、解析和数据转换。
- 已存在 MQTT 请求/响应 Topic 约定：`gateway/{gatewayName}/modbus/request/{requestId}` 与 `gateway/{gatewayName}/modbus/response/{requestId}`。
- 已存在请求超时记录和响应采集日志。

### 6.3 后续补齐需求

- 明确透传网关 Topic、Payload、Hex 格式、requestId 和 QoS 的正式协议文档。
- 完善 `WordOrder`、多寄存器类型和字符串类型解析规则，避免配置存在但运行时未完整应用。
- 完善转换规则，支持 Scale、Offset、EnumMap、BooleanMap、Clamp、Round 等常见规则。
- 完善采集任务校验，覆盖 SlaveId 范围、功能码范围、寄存器数量、数据类型与寄存器长度匹配、TargetName 必填等。
- 增加运行状态查询接口，例如每个网关调度器状态、待响应请求数量、最近采集时间、最近错误。
- 解决调度器启动和刷新策略中的并发与生命周期风险，确保后台服务不会因单个调度器阻塞而影响整体启动。
- 增加采集日志分页、状态过滤、任务过滤、点位过滤和时间范围过滤。
- 增加最小自动化测试，覆盖组帧、解析、转换、配置校验、请求超时和成功响应入库链路。

### 6.4 非功能需求

- 采集服务必须避免阻塞 MQTT 接入线程。
- 单个网关或单个点位异常不得影响其他网关或点位采集。
- 采集配置变更应能在合理时间内被后台服务刷新，目标不超过 30 秒。
- 采集日志必须保留足够排障信息，但应避免无限增长，后续需要归档或清理策略。
- 对现场网络不稳定、设备无响应、CRC 错误和网关离线要有明确状态。
- 第一阶段优先保证正确性、可诊断性和可维护性，不追求极限采集吞吐。
- 透传网关采集模块不得绕过设备租户和客户隔离。
- 采集值写入必须复用现有遥测发布链路，避免另建一套数据入库路径。

## 7. 验收标准

### 7.1 任务配置

- Given 已存在一个在线透传网关  
  When 创建包含从站和点位的采集任务  
  Then 系统应保存任务、从站、点位配置，并返回 `{ total: 1, rows: [task] }`。

- Given 采集任务 TaskKey 已存在  
  When 再次创建相同 TaskKey  
  Then 系统应返回重复错误，不应创建第二条任务。

- Given 点位功能码、寄存器地址或 TargetName 缺失  
  When 调用校验接口  
  Then 系统应返回明确校验错误。

### 7.2 调度与请求

- Given 采集任务已启用且网关在线  
  When 点位达到轮询周期  
  Then 后台服务应生成 Modbus 请求帧并发布到网关请求 Topic。

- Given 网关离线  
  When 点位达到轮询周期  
  Then 系统应跳过发送请求，并记录调试级或状态信息。

- Given 多个点位属于同一从站和相邻地址  
  When 调度器执行批次合并  
  Then 系统应尽可能生成合并批次，但不得破坏解析正确性。

### 7.3 响应处理

- Given 网关返回合法 Modbus 响应  
  When 系统匹配到 requestId  
  Then 系统应解析数据、应用转换规则、发布遥测并记录成功日志。

- Given 网关返回 CRC 错误或异常响应  
  When 系统处理响应  
  Then 系统应记录失败日志，不应写入错误遥测值。

- Given 请求超过超时时间无响应  
  When 超时检测触发  
  Then 系统应移除待响应请求并记录 Timeout 日志。

### 7.4 日志与诊断

- Given 调试人员查询采集日志  
  When 按网关、时间和状态过滤  
  Then 系统应返回分页日志列表。

- Given 某点位持续失败  
  When 查询日志  
  Then 日志应能看到请求帧、响应帧、状态、错误信息和耗时。

### 7.5 遥测写入

- Given 点位配置了目标设备和目标遥测键  
  When 采集成功  
  Then 目标设备的最新遥测和历史遥测链路应收到对应值。

## 8. 不在范围内

- 不在本模块实现边缘采集网关 JSON 上传。
- 不在本模块实现完整控制命令下发和写寄存器控制闭环。
- 不在本模块实现业务设备聚合视图。
- 不在本模块实现能耗统计。
- 不在本模块实现完整前端组态画面。
- 不在本模块实现所有工业协议，仅聚焦 Modbus 透传采集。
- 不在本模块处理 MQTT Broker 集群化问题。

## 9. 当前状态判断

**状态：已有第一版实现，需产品化补齐和稳定性强化。**

当前代码已经具备实体、API、调度、组帧、响应匹配、解析、遥测发布和日志记录的基础闭环。后续不应从零重写，应围绕协议文档、配置校验、转换规则、调度生命周期、运行诊断、测试覆盖和少量 API 一致性进行补齐。
